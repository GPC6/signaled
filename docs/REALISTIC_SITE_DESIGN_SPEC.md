# 시그널에듀 현실형 웹사이트 구조 명세

## 1. 목표

시그널에듀 웹사이트를 `메인 페이지 + 6개 핵심 페이지` 구조로 재정리한다.

각 번호는 독립된 한 페이지로 구성한다. 하위 항목은 별도 HTML 페이지로 이동하지 않고, 해당 페이지 안에서 탭 또는 스크롤 섹션으로 보여준다.

메인 페이지는 기존 방향처럼 이미지 캐러셀을 유지하되, 캐러셀 아래에 6개 핵심 페이지의 대표 내용을 요약적으로 보여주는 스크롤형 또는 탭형 섹션을 둔다.

## 2. 전체 사이트 구조

### 2.1 최종 페이지 맵

```text
/
├─ index.html                         # 메인
├─ signal-edu/index.html              # 1. 시그널에듀
├─ consulting/index.html              # 2. 컨설팅 프로그램
├─ ai-connects/index.html             # 3. AI 커넥츠
├─ partnerships/index.html            # 4. 학교·기관 연계
├─ stories/index.html                 # 5. 포트폴리오
└─ news-contact/index.html            # 6. 상담·소식
```

### 2.2 하위 페이지 처리 원칙

현재 존재하는 세부 HTML 페이지는 핵심 내비게이션에서 제거한다.

예:

```text
consulting/career-design.html
consulting/pathway-design.html
consulting/admission-strategy.html
consulting/learning-coaching.html
ai-connects/middle-score.html
ai-connects/student-record.html
ai-connects/literacy-dictionary.html
ai-connects/student-analysis.html
```

위 파일들은 당장 삭제하지 않아도 된다. 다만 공개 메뉴, 메인 카드, CTA 링크에서는 더 이상 직접 연결하지 않는다. 필요한 내용은 각 상위 `index.html` 안으로 흡수한다.

## 3. 공통 탐색 구조

대상 파일:

- `components.js`
- `main.js`
- `style.css`

### 3.1 상단 메뉴

상단 메뉴는 6개 핵심 페이지만 노출한다.

1. 시그널에듀
2. 컨설팅 프로그램
3. AI 커넥츠
4. 학교·기관 연계
5. 포트폴리오
6. 상담·소식

데스크톱에서는 6개 메뉴를 한 줄로 보여준다. 모바일에서는 햄버거 메뉴 안에 같은 6개 항목을 보여준다.
각 메뉴에 마우스를 호버하면 해당 메뉴의 세부 내역이 전체 드롭다운으로 들어간다. (아르떼 페이지 레퍼런스)
그리고 우측 햄버거 메뉴를 누르면 모든 메뉴의 세부 내역이 한번에 보여진다. 전체 드롭다운 시에는 뒤 페이지는 흐리게 보여진다.

### 3.2 페이지 내부 이동

하위 항목은 페이지 안의 탭 또는 스크롤 앵커로 제공한다.

권장 방식:

- 페이지 상단에 sticky 또는 고정형 서브내비를 둔다.
- 서브내비 항목 클릭 시 같은 페이지 안의 섹션으로 스크롤한다.
- 화면 폭이 좁은 모바일에서는 가로 스크롤 탭으로 보여준다.
- 별도 페이지 이동 또는 리다이렉트는 하지 않는다.

링크 예:

```text
signal-edu/#intro
signal-edu/#philosophy
signal-edu/#consultants
signal-edu/#location

consulting/#career
consulting/#pathway
consulting/#admission
consulting/#coaching
```

## 4. 메인 페이지

대상 파일:

- `index.html`
- `style.css`
- `main.js`

### 4.1 메인 첫 화면

기존처럼 이미지 캐러셀을 유지한다.

다만 현재처럼 `IMAGE PLACEHOLDER`가 보이면 실제 사이트처럼 보이지 않으므로, 캐러셀 이미지는 다음 중 하나로 교체한다.

- 실제 상담, 교육 프로그램, 강의, 활동 사진
- 실제 사진이 없을 경우 상담 리포트, 진로 로드맵, 학생 분석 자료를 보여주는 문서형 비주얼
- 서비스별 대표 이미지를 대신하는 편집형 정보 패널

캐러셀 카드 구성:

1. 시그널에듀
   - 대표 메시지: 학생을 이해하는 것에서 시작하는 교육
   - 링크: `signal-edu/`

2. 컨설팅 프로그램
   - 대표 메시지: 진로·진학·학습을 하나의 로드맵으로 설계
   - 링크: `consulting/`

3. AI 커넥츠
   - 대표 메시지: 학생 데이터를 상담 가능한 전략으로 연결
   - 링크: `ai-connects/`

4. 학교·기관 연계
   - 대표 메시지: 학교와 기관의 목적에 맞춘 교육 프로그램 운영
   - 링크: `partnerships/`

5. 포트폴리오
   - 대표 메시지: 합격소식과 활동 기록으로 확인하는 운영 결과
   - 링크: `stories/`

6. 상담·소식
   - 대표 메시지: 공지와 상담 안내, 오시는 길 확인
   - 링크: `news-contact/`

### 4.2 메인 요약 섹션

캐러셀 아래에 6개 핵심 페이지의 대표 내용을 뽑아 요약한다.

추천 UI는 두 가지 중 하나다.

1. 탭형
   - 상단에 6개 탭
   - 탭을 누르면 해당 페이지의 대표 하위 항목 2-3개와 CTA 노출
   - 한 화면에서 비교하기 좋다.

2. 가로 스크롤형
   - 6개 카드를 가로로 배치
   - 모바일과 잘 맞는다.
   - 실제 사이트처럼 자연스럽고 구현이 단순하다.

추천은 `가로 스크롤형 + 데스크톱 3열 보조 대응`이다. 탭보다 덜 인위적이고, 현재 사이트의 카드 구조를 크게 바꾸지 않아도 된다.

각 요약 카드 구성:

- 페이지명
- 한 줄 설명
- 대표 하위 항목 2-3개
- CTA

예:

```text
컨설팅 프로그램
진로·진학·입시·학습을 학생 상황에 맞게 설계합니다.
진로설계 / 대입컨설팅 / 학습검사·코칭
컨설팅 보기
```

### 4.3 메인에서 제거하거나 축소할 요소

- 근거 없는 숫자 카운트업
- 0으로 보이는 반응 수
- 과하게 많은 포트폴리오 이미지 그리드
- 실제 사진 없이 반복되는 `placeholder.svg`
- "추후", "구성할 수 있습니다", "확장됩니다" 같은 준비 중 문구
- 또한 전체적으로 모서리가 둥근 카드형태의 디자인은 피한다. (레퍼런스들을 참고하여 디자인 할 것)

## 5. 1번 페이지: 시그널에듀

대상 파일:

- `signal-edu/index.html`
- `signal-edu/signal-edu.js`
- `style.css`

### 5.1 페이지 목적

회사 소개, 교육철학, 컨설턴트, 오시는 길을 한 페이지 안에서 보여준다.

### 5.2 섹션 구조

```text
signal-edu/
├─ 소개              # #intro
├─ 교육철학          # #philosophy
├─ 컨설턴트          # #consultants
└─ 오시는 길         # #location
```

### 5.3 서브내비

페이지 상단 또는 히어로 아래에 다음 탭을 둔다.

- 소개
- 교육철학
- 컨설턴트
- 오시는 길

탭 클릭 시 같은 페이지 내 섹션으로 스크롤한다.

### 5.4 컨설턴트 정책

컨설턴트는 6-10명 범위로 정리한다.

권장:

- 실제 정보가 확인된 컨설턴트만 실명으로 노출
- 확인 전에는 개인 실명보다 역할 중심 카드 사용
- 조회수, 공감수, 상담수처럼 0으로 보이는 지표는 제거
- 사진이 없으면 `placeholder.svg` 대신 명함형 정보 패널 사용

## 9. 5번 페이지: 포트폴리오

대상 파일:

- `stories/index.html`
- `style.css`

### 9.1 페이지 목적

합격소식과 활동 갤러리를 통해 실제 운영 결과를 보여준다.

### 9.2 섹션 구조

```text
stories/
├─ 합격소식          # #acceptance-news
└─ 활동 갤러리       # #activity-gallery
```

### 9.3 서브내비

- 합격소식
- 활동 갤러리

### 9.4 합격소식 섹션

실제 학생 개인정보를 보호하면서 결과를 보여준다.

표시 방식:

- 연도
- 학교 또는 전형 정보
- 익명 처리된 학생 정보
- 준비 과정 요약
- 관련 프로그램

개인정보가 확인되지 않으면 구체적인 학교명이나 학생명은 넣지 않는다.

### 9.5 활동 갤러리 섹션

실제 사진이 있으면 사진 중심 갤러리로 구성한다.

실제 사진이 없으면 다음 구조로 대체한다.

- 활동명
- 대상
- 운영 방식
- 학생에게 남는 결과물
- 관련 문의 링크

## 10. 6번 페이지: 상담·소식

대상 파일:

- `news-contact/index.html`
- `data/news.json`
- `news-contact/news-detail.html`
- `news-contact/news-detail.js`
- `style.css`

### 10.1 페이지 목적

공지사항, 상담 안내, 오시는 길을 한 페이지에서 제공한다.

### 10.2 섹션 구조

```text
news-contact/
├─ 공지사항          # #notice
├─ 상담 안내         # #consultation
└─ 오시는 길         # #location
```

### 10.3 서브내비

- 공지사항
- 상담 안내
- 오시는 길

### 10.4 공지사항

`data/news.json`을 사용해 공지 목록을 보여준다.

공지 상세 페이지는 유지할 수 있다. 다만 상담·소식 페이지의 하위 구조 자체는 `공지사항 / 상담 안내 / 오시는 길` 세 섹션으로 유지한다.

### 10.5 상담 안내

확정된 상담 정보만 보여준다.

- 상담 대상
- 상담 방식
- 준비 자료
- 운영 시간
- 전화
- 이메일
- 방문 예약 안내

실제 카카오 채널 URL이 확정되지 않았다면 카카오 CTA는 숨긴다.

### 10.6 오시는 길

시그널에듀 페이지에도 오시는 길이 있지만, 상담·소식 페이지에서는 상담 신청 맥락에 맞춰 더 직접적으로 보여준다.

- 주소
- 방문 상담 예약 안내
- 주차 또는 대중교통 정보
- 지도 영역

실제 지도 연동 전에는 "지도 연동 예정" 같은 문구를 노출하지 않고, 주소와 방문 안내 중심으로 구성한다.

## 11. 디자인 톤

### 11.1 현실 사이트처럼 보이기 위한 원칙

- 실제 사진 또는 실제 문서처럼 보이는 비주얼을 사용한다.
- `IMAGE PLACEHOLDER` 문구는 공개 화면에 절대 노출하지 않는다.
- 0으로 보이는 지표, 임시 문구, 과장된 AI 홍보 문구를 제거한다.
- 카드 제목과 설명은 실제 상담자가 쓴 것처럼 구체적으로 쓴다.
- 한 섹션 안에서 같은 형식의 카드가 너무 많이 반복되지 않게 한다.

### 11.2 색상

현재 베이지 톤은 유지하되, 전체가 베이지 한 가지로 보이지 않게 한다.

- 기본 배경: 오프화이트
- 본문: 차콜 또는 먹색
- 보조 텍스트: 그레이 브라운
- 강조: 딥 네이비
- 보조 강조: 골드 브라운 또는 올리브 그린

### 11.3 이미지 정책

실제 사진이 있으면 실제 사진을 쓴다.

실제 사진이 없으면 다음 대체 비주얼을 쓴다.

- 상담 리포트
- 학생 분석표
- 진로 로드맵
- 프로그램 운영 체크리스트
- 합격소식 카드
- 활동 기록 카드

공개 화면에서 `placeholder.svg`를 그대로 노출하지 않는다.

## 13. 구현 계획

### 13.1 1단계: 정보 구조 정리

대상:

- `components.js`
- `index.html`
- 각 섹션별 `index.html`

작업:

- 상단 메뉴를 6개 핵심 페이지로 정리한다.
- 하위 메뉴는 각 페이지 내부 앵커로 바꾼다.
- 기존 세부 페이지로 직접 이동하는 링크를 상위 페이지의 해시 링크로 바꾼다.

검증:

- 상단 메뉴 클릭 시 6개 핵심 페이지만 열린다.
- 하위 항목 클릭 시 같은 페이지 안에서 스크롤된다.

### 13.2 2단계: 메인 페이지 개편

대상:

- `index.html`
- `style.css`
- `main.js`

작업:

- 이미지 캐러셀을 6개 핵심 페이지 기준으로 정리한다.
- 캐러셀 이미지 또는 대체 비주얼에서 `IMAGE PLACEHOLDER`를 제거한다.
- 캐러셀 아래에 6개 페이지 요약 카드 섹션을 추가하거나 기존 메뉴 안내 섹션을 교체한다.

검증:

- 메인 첫 화면에서 6개 핵심 페이지의 방향이 드러난다.
- 모바일에서 캐러셀과 요약 카드가 겹치지 않는다.

## 15. 검수 기준

개편 후 다음을 확인한다.

- 메인 + 6개 핵심 페이지 구조로 보인다.
- 상단 메뉴는 6개 핵심 페이지로 정리되어 있다.
- 각 번호별 페이지의 하위 항목은 같은 페이지 안에서 탭 또는 스크롤로 볼 수 있다.
- 하위 항목 때문에 별도 페이지로 리다이렉트되지 않는다.
- 메인에는 이미지 캐러셀과 6개 페이지 요약 섹션이 있다.
- 공개 화면에 `IMAGE PLACEHOLDER`가 보이지 않는다.
- 공개 문구에 `임시`, `추후`, `구성할 수 있습니다`, `확장됩니다`가 보이지 않는다.
- 컨설턴트 카드에 0으로 표시되는 조회수/공감수/상담수가 보이지 않는다.
- 모바일 390px 너비에서 가로 넘침이 없다.
- 데스크톱 1280px에서 헤더, 캐러셀, 요약 카드가 겹치지 않는다.
- 모든 주요 링크가 의도한 페이지 또는 같은 페이지 섹션으로 이동한다.
- 브라우저 콘솔에 JavaScript 오류가 없다.

## 16. 구현 전 확인할 항목

아래는 실제 구현 전에 사용자가 확정하면 좋다.

1. 메인 캐러셀에 사용할 실제 이미지가 있는지
2. 실제 이미지가 없다면 문서형 비주얼로 대체해도 되는지
3. 컨설턴트는 6명, 8명, 10명 중 몇 명으로 노출할지
4. 컨설턴트 실명을 그대로 사용할지, 역할 중심으로 바꿀지
5. 합격소식에 실제 학교명 또는 익명 사례를 사용할지
6. 카카오 상담 링크를 실제로 노출할지
7. 오시는 길 지도는 실제 지도 링크를 붙일지, 주소 안내만 먼저 둘지
